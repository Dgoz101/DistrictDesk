# DistrictDesk

**DistrictDesk** is a senior capstone project that streamlines IT helpdesk operations for school districts by centralizing ticket submission, tracking, and resolution. The system is designed around real-world IT workflows and stakeholder-driven requirements.

---

## Overview

School districts often rely on fragmented or outdated systems to manage technical support requests. DistrictDesk provides a centralized platform that improves visibility, accountability, and efficiency for IT staff while remaining simple and accessible for end users.

The project emphasizes maintainable architecture, clear role separation, and practical software engineering principles.

---

## Key Features

- Centralized IT ticket submission and tracking  
- Role-based access for staff, technicians, and administrators  
- Ticket status updates and resolution workflows  
- Organized, searchable request history  
- Scalable design suitable for district-wide use  

---

## Goals & Learning Outcomes

This project was developed as a senior-level capstone with the following goals:

- Apply software engineering best practices in a real-world scenario  
- Design a system based on stakeholder needs and feedback  
- Build a maintainable, extensible application architecture  
- Gain experience with full project lifecycle planning and execution
